# Лабораторная работа: Веб-игра «2048»

## Описание проекта

Полнофункциональная веб-версия популярной игры 2048, реализованная на чистом JavaScript, HTML5 и CSS3 без использования фреймворков. Игра представляет собой числовую головоломку-слайдер на сетке 4×4, где игрок объединяет плитки с одинаковыми числами для создания плитки со значением 2048 и выше.

##  Основные возможности

### Игровой процесс
- **Динамическая генерация поля**: сетка 4×4 создаётся программно через JavaScript с использованием `createElement()` и `appendChild()` (без `innerHTML`/`outerHTML`)
- **Движение плиток**: во всех четырёх направлениях (вверх, вниз, влево, вправо)
- **Слияние плиток**: при столкновении двух плиток с одинаковым значением они объединяются в одну с удвоенным значением
- **Подсчёт очков**: очки начисляются за каждое слияние (равны значению новой плитки)
- **Проверка окончания игры**: автоматическое определение отсутствия возможных ходов

### Дополнительные функции
- **Undo (отмена хода)**: возврат до 10 последних ходов с откатом счёта
- **New Game**: начать игру заново без перезагрузки страницы
- **Автосохранение**: состояние игры (поле, счёт, история) автоматически сохраняется в `localStorage` после каждого хода
- **Продолжение сессии**: после обновления или закрытия страницы игра продолжается с того же места
- **Таблица лидеров**: топ-10 лучших результатов с именами игроков и датами (хранятся в `localStorage`)

## Механика появления плиток

### Начало игры
При старте новой игры на поле появляется **1, 2 или 3 начальных плитки** (равная вероятность — 33.3% для каждого варианта).

### После каждого хода
После успешного хода на свободных клетках появляется:
- **1 плитка** — с вероятностью 50%
- **2 плитки** — с вероятностью 50%

### Значение новой плитки
Каждая новая плитка получает случайное значение:
- **«2»** — с вероятностью **90%**
- **«4»** — с вероятностью **10%**

Такое распределение соответствует оригинальной игре 2048 и обеспечивает сбалансированный игровой процесс: плитки «4» встречаются достаточно редко, чтобы не упрощать игру, но достаточно часто, чтобы добавить элемент непредсказуемости.

### Выбор позиции
Новая плитка появляется в **случайной пустой ячейке**. Все свободные клетки имеют равную вероятность быть выбранными.


## Реализация анимации

Для создания плавного и визуально привлекательного геймплея реализованы четыре типа анимаций:

### 1. Анимация движения плиток

Плитки плавно скользят в новые позиции за **0.2 секунды** при каждом ходе. Используется изменение CSS-свойств `left` и `top` с плавным переходом `ease-in-out`, что создаёт естественное ускорение в начале и замедление в конце движения.

### 2. Анимация появления новых плиток

Новые плитки появляются с эффектом **увеличения** (scale от 0 до 1) и плавного проявления (opacity от 0 до 1) за **0.15 секунды**. Это создаёт впечатление, что плитка «выпрыгивает» из пустой ячейки.

### 3. Анимация слияния плиток

При объединении двух плиток новая плитка **пульсирует**: увеличивается до **115%** от исходного размера и плавно возвращается обратно за **0.3 секунды**. Это создаёт яркий визуальный эффект, подчёркивающий момент слияния и увеличения значения плитки.

### 4. Анимация исчезновения плиток

Старые плитки при слиянии плавно **исчезают** с одновременным эффектом уменьшения (scale до 0) и затухания (opacity до 0) за **0.15 секунды**. Это создаёт ощущение, что плитки «схлопываются» и исчезают, освобождая место для новой объединённой плитки.

**Все анимации синхронизированы** и работают одновременно, создавая цельный и приятный визуальный опыт игры.
